# Contributing

欢迎改进这个项目。为了让协作更清楚，建议按以下方式提交改动。

## Workflow

1. 创建一个新分支。
2. 保持改动范围清晰，避免混入无关修改。
3. 如果修改了功能，尽量补充测试或说明。
4. 提交 Pull Request，并在描述中说明变更原因。

## Commit Style

建议使用简短明确的提交信息，例如：

```text
docs: update getting started guide
feat: add initial user module
fix: handle empty input
```

## Notes

如果项目后续确定了具体技术栈，可以在这里补充格式化、测试和发布流程。
